Fase 1 adalah **Pondasi**. Jika diibaratkan membangun rumah, ini adalah tahap pembersihan lahan, pembuatan pondasi semen, dan penyediaan material. Tanpa Fase 1 yang kuat, coding di Fase 2 akan berantakan.

Berikut adalah detail rencana **Fase 1 (Minggu 1)** untuk masing-masing anggota:

---

### ðŸ› ï¸ Tugas Anggota 1: Backend & Git Lead (Persiapan Sistem)

Kamu bertanggung jawab menyiapkan "rumah" digital tempat kalian bekerja.

1. **Setup GitHub:** Membuat repository dan mengundang Anggota 2 & 3 sebagai kolaborator.
2. **Struktur Folder:** Membuat folder proyek di `htdocs` dengan susunan yang rapi (`assets/`, `config/`, `includes/`, `modules/`, `uploads/`, `db/`).
3. **File .gitignore:** Membuat file `.gitignore` agar file sampah tidak ikut terunggah.
4. **Koneksi Database:** Membuat file `config/database.php` yang akan digunakan oleh semua fitur nantinya.
5. **Push Awal:** Mengirim semua struktur folder kosong ini ke GitHub agar bisa di-`pull` oleh temanmu.

### ðŸŽ¨ Tugas Anggota 2: Frontend Developer (Persiapan Tampilan)

Anggota 2 menyiapkan "alat rias" aplikasi agar tidak terlihat membosankan.

1. **Instalasi Bootstrap:** Mengunduh file Bootstrap dan meletakkannya di folder `assets/css/` dan `assets/js/`.
2. **Halaman Template:** Membuat file `includes/header.php` dan `includes/footer.php`. Ini gunanya supaya menu navigasi tidak perlu ditulis berulang-ulang di setiap halaman.
3. **Layout Utama:** Membuat kerangka halaman `index.php` (halaman utama) yang sudah terhubung dengan Bootstrap.

### ðŸ—„ï¸ Tugas Anggota 3: Database & Data (Persiapan Gudang)

Anggota 3 memastikan "gudang" penyimpanan data siap menampung informasi.

1. **Pembuatan Database:** Membuat database bernama `db_surat_smkn2` di phpMyAdmin.
2. **Eksekusi SQL:** Membuat tabel-tabel utama (`users`, `surat_masuk`, `surat_keluar`, `ref_kategori`).
3. **Input Referensi:** Memasukkan data awal ke tabel `ref_kategori` (misalnya: UND untuk Undangan, ST untuk Surat Tugas, dll) sesuai aturan sekolah.
4. **Backup SQL:** Mengekspor hasil database tersebut menjadi file `.sql` dan menyimpannya di folder `db/` proyek agar bisa dipakai oleh Anggota 1 & 2.
